# Session summary — 2026-05-14 — Capacitor wrapper + background geolocation

**Session A** from the multi-session plan at `sailline -docs/2026-05-14_race-tracking-improvements-plan.md`.

## What we worked on

Built the platform-adaptive geolocation layer required to keep GPS recording while the screen is locked — the root cause of Observation #3 from the Cook County Sailing test race.

The web/PWA path (today's production at `sailline.web.app`) is untouched and continues to work. A new Capacitor-wrapped Android build adds a foreground-service background watcher that survives screen lock.

## Decisions

| Decision | Rationale |
|---|---|
| **Android-only this session. No iOS native.** | User is on Windows; iOS native builds require macOS + Xcode. iOS users continue to use `sailline.web.app` (PWA) with the existing screen-on workaround. Adding iOS later is `npx cap add ios` from a Mac — no rewrite needed; the adapter abstracts platform branching. |
| **Plugin: `@capacitor-community/background-geolocation`** (MIT, free). | Limits cost ($0 vs. ~$300+ for Transistorsoft) and licence-debt. Actively maintained. If real-race testing surfaces drops, re-evaluate Transistorsoft. |
| **Adapter pattern in `lib/geolocation.js`** rather than platform branches inside `useTrackRecorder`. | Mirrors the regions.py ↔ regions.js mirror pattern in the codebase: one source-of-truth interface, two implementations. Keeps the hook pure and testable. |
| **Native plugin accessed via `window.Capacitor.Plugins.BackgroundGeolocation`** (Capacitor's global plugin registry), with `window.Capacitor` runtime detection. No `import` of any `@capacitor/*` module in the adapter. | The web bundle compiles and runs identically whether or not Capacitor packages are installed. **Initial implementation used `await import("@capacitor-community/background-geolocation")` — that failed `npm test` because Vite's import-analysis resolves dynamic imports at transform time, not runtime.** The registry pattern leaves nothing for the bundler to resolve. |
| **No iOS scaffolding generated.** | Generating `ios/` without a Mac to build it adds dead weight and confuses future readers. Will be added in a dedicated session. |
| **Skip `npx cap init`** in the runbook. | We pre-committed `capacitor.config.ts`. Running `cap init` would clobber it. |

## Files changed / added

### Added
- `frontend/src/lib/geolocation.js` — adapter. `createWatcher({onPosition, onError, options}) -> {stop()}`. Web path uses `navigator.geolocation.watchPosition`; native path lazy-loads `@capacitor-community/background-geolocation`. Exports `isNative()`, `normalizePosition()`, `DEFAULT_WATCH_OPTIONS`.
- `frontend/src/lib/geolocation.test.js` — vitest unit tests for the web branch + pure functions. Native branch is out of scope (no plugin in jsdom; on-device smoke covers it).
- `frontend/capacitor.config.ts` — `appId=com.sailline.app`, `appName=SailLine`, `webDir=dist`, `androidScheme=https`.
- `sailline -docs/2026-05-14-android-setup-runbook.md` — PowerShell-flavoured step-by-step: install deps, generate `android/`, edit `AndroidManifest.xml`, install on device, run the 15-minute screen-locked smoke test.

### Modified
- `frontend/src/hooks/useTrackRecorder.js` — swapped direct `navigator.geolocation.*` calls for `createWatcher(...)`. Removed local `WATCH_OPTS` constant (now `DEFAULT_WATCH_OPTIONS` in the adapter). Added `watcherPromiseRef` so `stop()` can wait for in-flight native `addWatcher` to resolve before tearing down. Point shape, flush behaviour, localStorage queue, breadcrumb state — all unchanged.
- `.gitignore` — added a Capacitor section. `frontend/android/` itself is committed (it's the native project); per-machine and build artefacts are ignored: `.gradle/`, `local.properties`, `build/`, `app/build/`, `**/*.keystore`, the synced `assets/public/` web bundle.

### Not yet on disk (require user action on Windows)
- `frontend/android/` — generated by `npx cap add android`. Steps in the runbook.
- `frontend/android/app/src/main/AndroidManifest.xml` — manual permission + foreground-service edits per the runbook.

## Tech debt flagged

| Item | Why it's debt | When to address |
|---|---|---|
| Native build not in CI | Cloud Build pipelines build the web app only. A regression in the native-only code path (Android manifest, plugin upgrade, Gradle config) won't be caught until a manual smoke test. | When iOS is added or when native code grows beyond manifest edits. Github Actions has macOS+Android runners; could add a smoke build job. |
| No iOS native shell | iOS users get the same screen-lock-pauses-tracking issue we have today. | Dedicated session from a Mac. Can run in any order after Session A is verified. |
| Capacitor branch of `geolocation.js` is not unit-tested | Plugin isn't installed in the jsdom test env. Branch is exercised only by on-device smoke test. | If the native branch grows non-trivial logic (e.g. permission state machine), add a mock-based test. For now the branch is ~30 lines of glue. |
| Permission downgrade is not surfaced in-app | If the user later switches Android Location permission from "Allow all the time" to "While using the app", background tracking silently stops mid-race. | Add a permission-status check + banner in a follow-up. Easy ticket. |
| Battery-optimiser exclusion is manual | OEMs (Xiaomi, OnePlus, some Samsung) aggressively kill foreground services. Currently documented in runbook only. | Detect via `PowerManager.isIgnoringBatteryOptimizations` and prompt the user once. Follow-up ticket. |
